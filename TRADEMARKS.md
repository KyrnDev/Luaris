This document covers use of project branding (name, logos, wordmarks, icons, and related visual identity), whether or not those marks are formally registered.

## Summary

- You may use the project name to describe factual compatibility or origin.
- You may not use the project branding in a way that suggests endorsement, affiliation, ownership, or official status if that is not true.
- You may not present forks or modified distributions as the original project.

## Allowed Uses

- "Built with Luaris"
- "Compatible with Luaris"
- Referencing this project in documentation, changelogs, or package metadata for factual attribution.

## Not Allowed Without Permission

- Using the project name/logo as the primary brand of a fork or republished package.
- Using the project logo/wordmark in product names, npm package names, domains, or marketing that implies official origin.
- Rebranding modified versions as if they are the original unmodified project.

## Forks and Modified Distributions

If you publish a fork or modified build:

- Use a distinct name.
- Clearly state that your distribution is a fork/derivative.
- Preserve original copyright and license notices.
- Do not imply it is the official project.
